# Classroom Seating Manager

Desktop app for teachers to visually manage classroom seating via drag-and-drop. Built with **Tauri 2 + React 18 + TypeScript**.

## Features

- Drag-and-drop seating grid ([dnd-kit](https://dndkit.com))
- Multiple courses and student rosters
- Google Drive sync (data roams with your teacher account)
- Google OAuth 2.0 PKCE login
- Local SQLite persistence (fully offline-capable)
- Windows + Linux support

### Google OAuth Setup

Drive sync requires a Google OAuth 2.0 Desktop app credential. You need a **Client ID** before running the app:

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a project → Enable **Google Drive API**
3. **Credentials** → **Create OAuth 2.0 Client ID** → **Desktop app**
4. Add redirect URI: `http://localhost:*`
5. Copy the generated **Client ID**

### Troubleshooting: stale Tauri build cache

If you get errors like:

```
error: failed to run custom build command for `classroom-seating-manager`
...
failed to read plugin permissions: .../permissions/app/autogenerated/commands/app_hide.toml: No such file or directory
```

The Rust build cache is corrupt. Tauri v2 generates permission files with hash-based directory names; moving the project or changing config between builds can cause hash mismatches. Fix:

```bash
cd src-tauri && cargo clean && cd ..
npm run tauri dev
```

The app reads `GOOGLE_CLIENT_ID` from the environment. If it's missing, you'll see:

> Google OAuth is not configured. Set GOOGLE_CLIENT_ID in your environment.

> **Tip for Linux/macOS:** Add `export GOOGLE_CLIENT_ID="..."` to your `~/.bashrc`, `~/.zshrc`, or shell config so you don't have to set it every time.
>
> **Tip for Windows (PowerShell):** Use `$env:GOOGLE_CLIENT_ID="..."` in your profile, or set it as a System Environment Variable.

## Architecture

**Local-first**: All writes go to local SQLite first. Drive sync is explicit (user-initiated push/pull). Full functionality works offline — login only gates sync.

**Sync flow:**
1. Mutations mark records as `dirty` in `sync_state`
2. Push: dirty records → serialize to JSON → upload to Drive
3. Pull: list Drive files → download → compare timestamps → merge
4. Conflict resolution: Last-Writer-Wins by `updated_at`

**Auth flow:** OAuth 2.0 PKCE — no client secret at runtime. Tokens stored encrypted in local SQLite.
